# SRSw — Software Requirements Specification

> Документ уточняет программные требования на основе SRS.

## 1. Конфигурация

- **SRSw-01**: Количество клавиш `NUM_KEYS = 12`.
- **SRSw-02**: Debounce = 20 ms.
- **SRSw-03**: Repeat = 200 ms.
- **SRSw-04**: Пауза между TX пакетами = 20 ms.
- **SRSw-05**: Keepalive = 1000 ms, payload `0x1F`.

## 2. Требования к keyboard logic

- **SRSw-KB-01**: Опрос клавиш должен выполняться в главном цикле.
- **SRSw-KB-02**: При переходе в стабильное состояние `pressed` генерируется событие нажатия.
- **SRSw-KB-03**: При отпускании активной клавиши генерируется событие отпускания.
- **SRSw-KB-04**: Автоповтор генерирует периодические события нажатия активной клавиши.
- **SRSw-KB-05**: Одновременно активной считается первая обнаруженная нажатая клавиша.

## 3. Требования к UART TX

- **SRSw-TX-01**: Отправка не должна нарушать минимальную паузу `tx_pause_ms`.
- **SRSw-TX-02**: При наличии payload отправляется payload.
- **SRSw-TX-03**: При отсутствии payload и достижении таймаута отправляется keepalive.

## 4. Требования к UART RX/LED

- **SRSw-RX-01**: RX-декодер должен принимать 1-байтовый пакет.
- **SRSw-RX-02**: Биты [4:0] определяют номер LED в диапазоне 1..12.
- **SRSw-RX-03**: Биты [7:6] определяют состояние (0 = off, non-zero = on).
- **SRSw-RX-04**: Некорректные LED-индексы должны игнорироваться.

## 5. Требования к concurrency

- **SRSw-CON-01**: Доступ к ring buffer из контекстов с разным приоритетом должен защищаться критической секцией.
- **SRSw-CON-02**: RX mailbox HAL должен читаться атомарно.

## 6. Требования к качеству

- **SRSw-QA-01**: Для core-модулей должны быть unit-тесты.
- **SRSw-QA-02**: CI должна запускать unit-тесты и сборку firmware.
