# Configuration Management

## 1. Идентификация конфигурационных единиц (CI)

- Код: `src/`, `include/`, `tests/`.
- Сборка: `Makefile`.
- CI/CD: `.gitlab-ci.yml`.
- Требования/документация: `docs/*.md`, `README.md`.

## 2. Версионирование

Рекомендуется SemVer для релизов:
- MAJOR: несовместимые изменения протокола/API;
- MINOR: функциональные добавления;
- PATCH: исправления без изменения контракта.

## 3. Baseline

Каждый релиз должен фиксировать:
1. commit SHA;
2. версию релиза;
3. версию toolchain (gcc/avr-gcc);
4. результаты `make test` и `make avr`;
5. список документов `docs/*`, входящих в baseline.

## 4. Управление изменениями

- Изменения требований: через MR с обновлением SRS/SRSw/Traceability.
- Изменения кода: через MR с тестами.
- Изменения интерфейсов: обязательное обновление ICD.

## 5. Воспроизводимость

- Локально: `make test`, `make avr`.
- CI: pipeline jobs `unit_tests`, `build_firmware`.
- Артефакты: `.elf`, `.hex`.
