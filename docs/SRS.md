# SRS — System Requirements Specification

## 1. Назначение

Прошивка реализует USB/UART-совместимую клавиатурную логику для AVR 8-bit (ATmega128A) с:
- опросом 12 кнопок;
- подавлением дребезга;
- поддержкой автоповтора активной клавиши;
- передачей событий по UART;
- приёмом команд управления подсветкой (LED) по UART;
- периодическим keepalive-пакетом.

## 2. Бизнес-цели

1. Стабильное и предсказуемое поведение клавиатуры в embedded-среде.
2. Минимальная латентность реакции на нажатие.
3. Простая сопровождаемость и тестируемость.
4. Воспроизводимая сборка через CI/CD.

## 3. Границы системы

Входит:
- логика клавиатуры и протокол UART;
- драйверная интеграция с GPIO/UART/Timer;
- unit-тесты core-логики.

Не входит:
- bootloader/OTA;
- USB-стек;
- desktop-утилиты конфигурации.

## 4. Функциональные системные требования

- **SRS-F-01**: Система должна обрабатывать 12 кнопок (индексы 0..11).
- **SRS-F-02**: Должен применяться debounce для подавления ложных срабатываний.
- **SRS-F-03**: Должен поддерживаться автоповтор активной клавиши.
- **SRS-F-04**: События нажатия/отпускания должны отправляться по UART в бинарном пакете.
- **SRS-F-05**: При отсутствии событий должен отправляться keepalive с заданным периодом.
- **SRS-F-06**: Входящие UART-команды LED должны применяться к 12 каналам подсветки.

## 5. Нефункциональные требования

- **SRS-NF-01**: Решение должно быть модульным (HAL + core), без жёсткой сцепки с регистрами в бизнес-логике.
- **SRS-NF-02**: Core-часть должна быть кросс-тестируема на host (gcc).
- **SRS-NF-03**: Проект должен собираться в CI и выдавать firmware artifacts.
- **SRS-NF-04**: Конфигурация сборки и тестов должна быть документирована.

## 6. Ограничения

- Платформа: AVR 8-bit.
- Целевой МК по умолчанию: ATmega128A.
- Тайминги зависят от частоты `F_CPU`.
